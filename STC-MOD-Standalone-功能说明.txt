STC-MOD-Standalone 功能说明
===========================

本文档只说明 STC-MOD-Standalone 这个 MOD 工具包含哪些功能。
安装、更新、PM2、回滚等操作指令请看桌面上的：
STC-MOD-Standalone-云端使用说明.txt


一、这个项目是什么
------------------

STC-MOD-Standalone 不是一个单独运行的新酒馆。

它是一个给官方 SillyTavern 使用的 MOD 安装与更新工具。

它的作用是：

1. 自动下载官方 SillyTavern。
2. 把 STC-MOD 后端功能接入 SillyTavern。
3. 把 STC Admin Panel 前端扩展接入 SillyTavern。
4. 自动应用必要的核心补丁。
5. 保留官方 SillyTavern 的更新能力。
6. 支持更新失败后回滚到上一版本。

简单说：

官方 SillyTavern 负责原本的聊天、角色卡、模型接口等功能。
STC-MOD 负责增加用户管理、注册、邀请码、公告、论坛、公共角色库、API 密钥保险箱、存储限制、系统监控等功能。

三、云端一键安装与更新工具
--------------------------

这是 STC-MOD-Standalone 最外层的工具能力。

包含文件：

install.sh
update.sh
rollback.sh
status.sh
scripts/install.mjs
scripts/release-common.sh

主要功能：

1. 自动 git clone 官方 SillyTavern。
2. 自动把 MOD 文件复制到新下载的 SillyTavern 里面。
3. 自动安装依赖。
4. 自动生成 releases/current/shared 结构。
5. 支持以后更新官方 SillyTavern。
6. 支持更新后重新应用 MOD。
7. 支持更新失败不切换版本。
8. 支持回滚到上一版本。

安装后的典型目录：

~/stc-app/
  current -> releases/某个版本
  releases/
  shared/
    config/config.yaml
    data/
    plugins/
    extensions/
  start.sh

这个结构的意义：

current 是当前正在运行的 SillyTavern。
releases 里面保存每次安装或更新出来的版本。
shared 保存配置和用户数据。

更新 SillyTavern 时，新的官方代码会放进新的 release 目录。
用户数据不会被覆盖。
如果新版本出问题，可以 rollback 回旧 release。


四、STC 后端模块
----------------

后端模块目录：

src/stc-mod/

它是整个 MOD 的核心。

它通过 SillyTavern 的 server-main.js 注入几个钩子：

1. 加载 STC-MOD 模块。
2. 给 STC-MOD 自定义接口提供 CSRF 豁免。
3. 注册公开页面路由。
4. 注册公开 API。
5. 注册需要登录的私有 API。

主要入口文件：

src/stc-mod/index.js

主要导出能力：

shouldSkipCsrf(req)
setupPublicRoutes(app)
setupPublicApi(app)
setupPrivateRoutes(app)

意思是：

STC-MOD 后端不是完全重写 SillyTavern。
它是挂在 SillyTavern 服务器上的外挂模块。


五、STC Admin Panel 前端管理面板
-------------------------------

前端扩展目录：

public/scripts/extensions/third-party/stc-admin-panel/

这是一个 SillyTavern 第三方扩展。

它主要提供一个图形化管理面板，让管理员或用户可以在网页里操作 STC-MOD 的功能。

常见功能入口包括：

1. 用户管理
2. 邀请码管理
3. 公告管理
4. 邮件配置
5. OAuth 配置
6. 存储空间管理
7. 系统监控
8. API 密钥保险箱
9. 默认用户模板
10. 公共角色卡库
11. 论坛相关管理

注意：

这个前端扩展单独复制到官方 SillyTavern 里没有意义。
它需要后端的 /api/stc/* 接口一起存在，才能正常工作。

十五、API 密钥保险箱
-------------------

相关文件：

src/stc-mod/services/privacy-vault.js
src/stc-mod/routes/private/privacy-vault.js
patches/core-files/src/endpoints/secrets.js
patches/core-files/public/scripts/secrets.js

主要功能：

1. 保护用户填写在 SillyTavern 里的 API Key。
2. 启用后，把 secrets.json 里的 API Key value 加密保存。
3. 使用 AES-256-GCM 加密。
4. 解锁后，服务端只在内存中短期保存解密密钥。
5. 锁定状态下，读取已加密 API Key 会失败并提示解锁。
6. 支持第一次启用时迁移已有明文 API Key。
7. 支持忘记密码后重置保险箱。

用户界面能力：

1. 未启用时提示启用保险箱。
2. 已锁定时提示解锁。
3. 已解锁时允许正常保存和使用 API Key。
4. 忘记密码时可以重置保险箱。

重要说明：

保险箱密码不是账号登录密码。
如果忘记保险箱密码，已加密的 API Key 无法恢复。
重置保险箱会清理已加密的 API Key，需要重新填写。

十九、官方账户备份按钮隐藏补丁
------------------------------

相关文件：

patches/core-files/public/scripts/user.js

这个补丁只做一件事：

普通用户打开官方账户资料页面时，隐藏官方的“下载备份”按钮。

当前设计：

1. Admin 管理员仍然能看到并使用下载备份按钮。
2. 普通用户只是在前端看不到按钮。
3. 后端备份接口没有被禁用。
4. 其他用户仍然可以正常使用账号和数据。
5. 这不是安全权限限制，只是隐藏按钮。

为什么这样做：

你的需求是“其他用户也要备份，只是不让他们下载，只要隐藏按钮”。
所以这里只改前端显示，不额外加后端限制。


二十、静态资源缓存优化
----------------------

相关文件：

patches/core-files/src/server-main.js

主要功能：

1. 给 JS、CSS、字体、图片等静态文件增加缓存。
2. 减少重复下载。
3. 改善云服务器或国内网络环境下的加载速度。

缓存策略：

JS、CSS、字体、图片等静态资源缓存 1 天。
HTML 页面缓存 1 小时并保留协商缓存。

作用：

登录后 SillyTavern 主界面会加载很多 JS、CSS 和图片资源。
缓存后，第二次打开会更快。

二十六、这个 MOD 对官方 SillyTavern 的影响
------------------------------------------

STC-MOD 尽量把新增功能放在 src/stc-mod/ 和前端扩展目录里。

但是有些功能必须改官方核心文件，例如：

1. server-main.js
2. secrets.js
3. avatars.js
4. personas.js
5. user.js

这些核心文件补丁放在：

patches/core-files/

为什么要改核心文件：

1. 自定义登录页、欢迎页需要接入官方服务器路由。
2. API 密钥保险箱需要接管 secrets 的读写逻辑。
3. 头像和 persona 相关功能需要适配 MOD 行为。
4. 账户备份按钮隐藏需要改官方用户页面脚本。

所以这个 MOD 不是完全零侵入。
但它已经尽量把改动集中起来，方便以后维护。


二十七、更新时功能如何保持
--------------------------

更新流程不是直接进入酒馆目录 git pull。

正确方式是用 STC-MOD-Standalone 的 update.sh。

update.sh 会：

1. 下载或更新官方 SillyTavern。
2. 创建新的 release 目录。
3. 重新复制 STC-MOD 后端模块。
4. 重新复制 STC Admin Panel 前端扩展。
5. 重新应用 core-files 补丁。
6. 安装依赖。
7. 检查关键文件语法。
8. 成功后才切换 current 到新版本。

这样更新后，MOD 会重新生效。

如果新版本不兼容，旧版本还保留在 releases 里。
可以用 rollback.sh 回滚。


二十八、回滚能力
----------------

rollback.sh 的作用：

1. 把 current 切回上一个 release。
2. 不删除 shared 用户数据。
3. 不覆盖 config.yaml。
4. 不破坏用户聊天数据。

适合场景：

1. 官方更新后 MOD 不兼容。
2. 更新后网页打不开。
3. 更新后登录异常。
4. 更新后某些功能报错。

回滚只是切换代码版本，不是恢复数据库快照。
如果用户数据已经被新版程序写坏，仍然需要单独的数据备份。


二十九、不包含什么
------------------

这个 MOD 不包含以下内容：

1. 不包含模型 API 服务。
2. 不包含 OpenAI、Claude、Gemini 等平台的 API Key。
3. 不负责给服务器安装系统环境以外的软件服务。
4. 不保证永远兼容所有未来官方 SillyTavern 版本。
5. 不会自动解决官方大改核心文件造成的冲突。

如果官方 SillyTavern 未来大改 server-main.js、secrets.js、user.js 等文件，
MOD 补丁可能需要手动维护。


三十、适合的使用场景

这个 MOD 适合：

1. 云端部署 SillyTavern。
2. 多用户共用一个酒馆。
3. 需要注册和邀请码。
4. 需要管理员后台。
5. 需要公告、论坛、公共角色卡库。
6. 需要限制用户存储。
7. 需要保护用户 API Key。
8. 需要官方更新后 MOD 能重新应用。
9. 需要更新失败可以回滚。

不太适合：

1. 只在本机单人使用酒馆。
2. 不需要注册、不需要多用户、不需要后台。
3. 希望完全不改官方任何文件。
4. 希望直接把 MOD 当成独立网站运行。


三十一、最简总结
----------------

STC-MOD-Standalone 做了两件事：

第一，它给 SillyTavern 增加了一套云端多人运营功能。

包括：

注册、邀请码、公告、邮件、OAuth、论坛、公共角色库、用户扩展、存储限制、API 密钥保险箱、系统监控、默认模板、后台管理面板。

第二，它把这些 MOD 功能做成了可安装、可更新、可回滚的云端工具。

也就是：

可以自动安装官方 SillyTavern。
可以自动应用 MOD。
可以以后更新官方 SillyTavern。
可以更新失败后回滚。
可以配合 PM2 长期运行。
